Welcome to visit my sketch gallery: DORAEMON'S WORLD

Personal perspective toward this Anime: Doraemon's props represent material, environment and exterior. Nobita, on the other hand, represents the spirit, the ego, the inner. In Doraemon, the two are in harmony. The dreams doraemon brings to children include both the outer form of exploring the infinite possibilities of technology and the inner form of breaking through his deeply rooted personality. This kind of breakthrough is difficult, and we may not be able to change it for a long time. But Fujiko F. Fujio believes that at least we can live with this goal actively, so that our hearts will never be empty, so that the hope of life will always illuminate the road ahead for us. Besides, a little effort is better than none at all.
How I finished the sketch and reflection:The values stored in my lists are mainly strings and integers which represent the color parameters, scale parameters and rotation angles. Initially, I decided to directly loop through them or call them with index. However, considering the randomness of the sketch, I decided to put list variables inside the random function to generate random elements in lists following each iteration.Worth mentioned that I used translate function inside each of the user defined function and passed the coordinate parameters of these functions into the translate function. Therefore I can decide the position about this part of the sketch out of the function scope. Another reason for translate has been mentioned above, it facilitates my work about developing the sketch with more clear coordinates that start from (0, 0). User-defined functions helped me to better organize the whole project. From my perspective, this serves a similar function as oriented object programming. In detail, It allows us to separate the project into different specific part, then leave the main logic line clear and concise. I mainly passed colors, x coordinates, y coordinates, rotation angles, and scale size into my functions. These parameters was first passed into the draw creature function and shared by others altogether. This allows us to change the attributes of the imaginary creature only by revising elements inside the function brackets.
